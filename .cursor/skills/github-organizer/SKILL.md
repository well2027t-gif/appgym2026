# GitHub Organizer

## Objetivo

Manter o projeto organizado para evolução segura.

## Instruções

1. Use `main` como branch principal, salvo instrução contrária.
2. Sugira branches curtas e descritivas:
   - `feat/home-layout`
   - `fix/profile-storage`
   - `chore/coach-base`
3. Prefira commits pequenos e objetivos.
4. Nunca use ações destrutivas sem confirmação.
5. Ao preparar PRs, resuma:
   - o que mudou
   - por que mudou
   - como testar

## Modelo de commit

Use mensagens curtas como:

```text
feat(home): adicionar base do coach tai
fix(profile): salvar nome e peso no storage
chore(github): documentar fluxo inicial do projeto
```

## Checklist

- Branch correta
- Mudanças isoladas
- Build passando
- Resumo de teste pronto
